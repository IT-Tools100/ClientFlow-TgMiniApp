# AGENTS.md

## Роль Codex

Ты работаешь как senior frontend/fullstack-разработчик и технический архитектор проекта **ClientFlow Mini App**.

Твоя задача — поэтапно создать Telegram Mini App для учета клиентов, заявок, задач и сделок малого бизнеса.

## Главные принципы

1. Делай проект пошагово.
2. Не усложняй раньше времени.
3. Сначала создай качественную frontend-оболочку на mock data.
4. Supabase подключай только после готовой UI-структуры.
5. Код должен быть чистым, понятным и компонентным.
6. Каждый этап должен завершаться проверкой `typecheck`, `lint`, `build`.
7. После каждого этапа обновляй `TASKS.md`.
8. Не удаляй важные файлы без явной необходимости.
9. Не используй платные сервисы, кроме бесплатных тарифов Supabase/Vercel.
10. Проект должен выглядеть как реальный коммерческий кейс для GitHub-портфолио.

## Стек

- Next.js
- TypeScript
- Tailwind CSS
- Supabase
- Telegram Mini Apps SDK / Telegram WebApp API
- Vercel
- GitHub

## Стиль проекта

Проект должен иметь premium mobile-first интерфейс в стиле **Liquid Glass**:

- glassmorphism cards
- blur
- soft gradients
- glowing accents
- thin borders
- readable typography
- mobile-first layout
- Telegram Mini App feeling

## Ограничения

На первом этапе нельзя:

- подключать Supabase
- делать сложную авторизацию
- подключать платежи
- делать командные роли
- добавлять AI-функции
- делать сложный backend

Сначала нужен сильный UI shell и mock data.

## Качество

Код должен быть готов к публичному GitHub-репозиторию.

Не оставляй мусорные комментарии, временные файлы, неиспользуемые импорты и неаккуратные названия.
